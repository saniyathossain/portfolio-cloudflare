# 40 — PageSpeed 100 (desktop + mobile) + glass-in-every-detail + responsive/tablet parity + safe motion polish

## Context
Three problems, one batched deploy, executed here:
1. **PageSpeed isn't a perfect 100** on desktop *or* mobile (mobile is worse — 4× CPU throttle + slow-4G in the Lighthouse lab). Cost-centers: render-blocking CSS/font, network-gated interactivity, an unused 65 KB Motion One bundle, always-on canvas loops.
2. **Mobile/tablet design is weaker than desktop** — no real tablet tier (`768px` resolves almost everywhere to the phone branch), binary (non-fluid) spacing steps, hover-only glass/motion that dies on touch, sub-44px targets, hidden content, iOS input zoom, popover overflow, and a latent CSS-specificity bug that silently disables the intended desktop H1 sizing.
3. **Liquid glass isn't applied to every surface** — several controls (modal inputs, social links, outline buttons, tooltips, stats panel, footer CTA) are flat/plain next to the reference glass primitives (`.glass-pill`, `.brand-pill`, `.edu-row`).
4. Richer **micro + mouse animation**, locally vendored, with **zero PageSpeed cost**.

Hard constraints: **100/100 both form factors**; no UI/UX regression; light-only theme; every animation `prefers-reduced-motion`-gated; no new default-build network requests; no inline `style=""`; `portfolio.json` is the content source; `<head>` generated by `scripts/sync-head.js` (never hand-edit `SYNC:*` markers); `./build-css.sh` only if new Tailwind utilities are added; Cloudflare free plan → **batch everything into one build + one deploy**.

Five read-only audits are complete and drive every section below: render/LCP, JS/TBT, responsive/tablet, Cloudflare settings, and **two follow-up audits** (glass-surface completeness, mobile/tablet aesthetic parity) plus a **web research pass** on performance-safe glass/motion techniques.

## D. Motion / micro-interaction polish — corrected by performance research
Research verdict: **don't add a smooth-scroll JS library at all** (Lenis/Locomotive fight native inertia + add main-thread weight). Use the zero-JS, compositor-only replacement instead.

1. **Hand-rolled transform-lerp cursor** (`public/assets/js/cursor.js`, ~2-3KB): single `mousemove` → rAF loop lerping `cur += (target-cur)*0.15`, writes only `transform: translate3d()`; **self-cancelling** (stop rAF when `|target-cur| < ε`, restart on next move — avoids idle repaint, per the 14islands reference). Magnetic pull toward interactive elements on `pointerenter`. Gate: `pointer:fine` + `!reduced`, desktop-idle-loaded via `boot.js` exactly like aurora/liquid-hero.
2. **CSS scroll-driven one-shot reveals** (`animation-timeline: view()`) replacing any scroll-JS ambition — runs off the main thread, ~85% baseline support, wrap in `@media (prefers-reduced-motion:no-preference)`, no-op fallback (final state via `animation-fill-mode`), **no polyfill**. Use for: panel/service-row/skill-card "wet sheen" sweep on first scroll into view (ties into E's touch-glass restoration below).
3. **View Transitions API** (optional, small scope): feature-detect `document.startViewTransition`, use only for a couple of same-document state swaps (e.g. modal open/close) — Baseline since Oct 2025, near-zero cost, graceful no-op elsewhere.
4. **Mobile "premium" swaps** (no JS cost): `:active { transform: scale(0.96) }` + `touch-action: manipulation` on all primary controls; optional `navigator.vibrate(10)` on key CTA taps (progressive enhancement only, no iOS support).
5. **CSS-first micro-interactions:** `transform`/`opacity`-only hover/press/focus, `<300ms`, ease-out — extend to any control currently missing an active/touch state.

**AVOID:** Lenis/Locomotive/GSAP+ScrollTrigger (main-thread weight, fights native scroll), Motion One (removed in A3), blobity/cursor-effects/kursor (canvas TBT), SVG-refraction "real" Liquid Glass via `backdrop-filter:url(#filter)` (Chromium-only, paint-storm on resize), Lottie/Rive, Splitting.js/SplitType (skip — no confirmed need, keep footprint minimal).

## E. Liquid glass in every detail (new — from the glass-completeness audit)
Reference glass recipe = frost fill + top light line + tinted rim + specular gloss film + soft shadow (`.glass-pill`, `.brand-pill`, `.edu-row` are full-strength). Bring the weakest surfaces up **without adding new `backdrop-filter` layers** (CSS-only: gradients, inset highlights, mask-rim, existing `.sheen`/`::before` gloss groups).

Ranked, lowest-cost-highest-impact:
1. **`.modal-field input/textarea`** (`:2925`, weakest surface in the system) — add `inset 0 1px 0 rgba(255,255,255,.9)` + a `color-mix(--tint)` focus rim.
2. **`.social-link`** (`:652`) — join the gloss `::before` group (`:1828-1855`) + copper rim + glow, matching `.back-to-top`.
3. **`.pill-btn-outline` (About) + footer `.pill-btn.bg-surface`** — add a gloss `::after` (copy `.pill-btn-glass::after` `:1391`) + inset highlight.
4. **`.stats-panel`** (`:2100`) — append `var(--glass-highlight)` to its shadow + a soft-light top film `::after` (like `.glass-panel::after` `:328`).
5. **`.popover__panel` + `.brand-tooltip` + `.partner-orb__tip`** — add a faint diagonal gloss + tint rim (unifies floating chrome); combine with B8's clamp fix.
6. **`.exp-row__period` + `.edu-place`/`.edu-row__place`** — swap neutral rim for `color-mix(--tint)` to match `.edu-row__years` (`:474`).
7. **Shared tinted hairline** replacing flat `var(--line)`/`border-white/10` at about-footer divider, service/exp separators, footer borders.
8. **`.modal-panel`** (`:2863`) — add `--glass-highlight` top-light + tint rim.
9. **Scrollbar thumb** (`:203`, desktop `hover:hover`-gated only — zero mobile cost) — tinted gradient + inset highlight.
10. Scroll-triggered one-shot `.sheen` (via D2's `animation-timeline:view()`) on glass panels/cards so touch devices see the "wet" gloss without needing hover/pointer — the biggest single fix for "mobile glass looks flat" per the parity audit.
11. `.hero__glow` is fully `display:none` below 1024 (`:3052`) — restore a **static** pre-softened radial (no `filter:blur()`, no animation) so mobile keeps a light source within budget.
12. Success-check circle, `.uc-panel` body, `.pill-btn__icon` — extend the black+copper gloss recipe / `--frost-bg` respectively.

All of E is paint-once CSS (no continuous cost) — verified against the research audit's "ration `backdrop-filter`, prefer gradients+insets+baked grain" verdict.
