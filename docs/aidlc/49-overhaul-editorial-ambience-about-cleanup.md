## W1 — AIDLC doc 48 + code conventions (do first: it governs all other edits)

**New doc `docs/aidlc/48-code-conventions-template-literals-early-returns.md`** codifying two rules for all hand-written `.js`:
- **Template literals** over string concatenation: `` `${a}b` `` not `a + "b"`.
- **Early returns / guard clauses**, **SOLID**, and **simple readable code** over clever/nested code.

**Implement across the JS** (representative sites the audit found — sweep all, not just these):
- `app.js` — clock build `h+":"+m+mer` (≈329), date (≈331), `"#"+id` (≈401), style string `"--tint:"+v.tint+...` (≈603), swap classes `"is-swapping-"+dir_` (≈635/654/658).
- `data.js` — hex build `"#"+to(r)+...` (≈175), `yearStr+" "+monthStr` (≈333), `"mailto:"+email` (≈411).
- `aurora.js` — `"rgba("+b.h+...` (≈64–66); `blur-reveal.js` — hl-class concat (≈88); `boot.js` — `src+"?v="+ASSET_V` (≈18/24); `editorial.js` — `'Nothing for "'+q+'"...'` (≈86), `x+"px"` style writes throughout.
- Keep changes mechanical/behaviour-preserving; run `node --check` on each file; do not touch `*.min.js` by hand (regenerated by build).

Guardrail: this is a readability refactor — **no behaviour change**, verified by build + smoke test.
